* Introduction
  APDL (Ansys Parametric Design Language) is the solver scripting
  language of the FEA (Finite Element Analysis) suite [[http://www.ansys.com][Ansys]] (ANalysis
  SYStem, registered TM).

  APDL-Mode includes features for inspecting, writing and debugging
  snippets, scripts and whole model files.  APDL-Mode enables, in
  conjunction with the GNU-Emacs editor, an advanced APDL environment
  with features like solver communication (GNU-Linux only), license
  reporting, keyword completion, code templates, dedicated
  keybindings, etc.

  # TODO: info sufficient? Please convince yourself with the APDL-Mode in-depth
  # documentation

  With the advent of the modern Ansys FEA GUIs - like `WorkBench' and
  `Discovery' - the usage of APDL as a comprehensive modelling
  language has diminished.  But APDL is here to stay: `WorkBench'
  controls exclusively the Ansys solver with it!  It is sending APDL
  input (.dat) files for creating the model and controlling the
  solver.  For a true understanding of the Workbench modelling process
  the study of its generated APDL code is prerequisite.

  Furthermore the WorkBench GUI does not support all solver features.
  So APDL `Command Snippets' enhance WorkBench's modelling
  capabilities.
* COMMENT Some features
  Nowadays I find APDL-Mode mostly useful in writing these `Command
  Snippets' or reading Snippets from others.  For in-depth
  investigations APDL-Mode is very convient to study the solver (.dat)
  files.
** In-built APDL command help with argument counter
   Especially for commands with a large number of arguments it is
   cumbersome to count the arguments, *C-?* facilitates this for you
   and visualises dynamically at which parameter position your cursor
   currently is.
